QUICK ANSWERS
-------------
Q: Two interfaces, same abstract method, one class implements both?
A: Implement ONCE in class. Call on child → your single method runs. (I03, I05)

Q: Two interfaces, same default method?
A: Compiler forces override in implementing class. (I04)
   Call child.method() → your override runs. (I05)
   Inside override: A.super.method() and B.super.method() to call each default. (I06)

Q: Multiple class inheritance?
A: Not allowed. (I01, I02)

Q: Diamond problem?
A: Class can't extend Tiger and Lion both. Use interfaces instead. (I02)

Q: Field with same name in parent/child?
A: Field HIDING — reference type decides which field. (I09)

Q: static method same signature in parent/child?
A: Method HIDING — not polymorphic. (I08)
